Detailed Description

User interface.

Todo:
Write the description of the effectively implemented user interface.

The user interface is implemented as a Finite State Machine.

The states are:
WHITE, AMBER, RED, GREEN, BLUE, IDLE, STOP, START

The events and transitions are:

Touchgecko pressed
Go to state START, initialize the system and switch to state IDLE.
Touchslider touched
Adjust value of the currently active state.
Pushbutton 0 pressed
Go one state to the right, wrap around from IDLE to WHITE.
Pushbutton 1 pressed
Go one state to the left, wrap around from WHITE to IDLE.
Remote command from serial interface received
The received string is parsed and the new state and value set accordingly.

Any changes in state or value are reflected on the display and also sent over the serial interface to the remote control.

◆ UI_FSM_event_RemoteControl()

void UI_FSM_event_RemoteControl ( void  )

Part of the user interface finite state machine: Remote control events.

If a string is available from the serial interface read and analyze it.
The core is a parser which analyzes the received command string.
It extracts the state and the value (if any). The value is separated by a ' ' from the state.
There are 3 possible outcomes:
- No valid state is recognized => nothing happens
- A valid state is recognized, but no valid value => switch to that state, don't change the value
- Valid state and value received => switch to that state and change the value
