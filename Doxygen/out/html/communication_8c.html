Detailed Description

Simple asynchronous communication with just a string as second level buffer.

The code is inspired by AN0045 USART or UART Asynchronous mode and by AN0017 Low Energy UART
It uses only the HW-buffers in connection with TX- and RX-interrupts.
Overflow handling is not implemented as processing is faster than the baud rate.

Note
As the LEUART runs on a low frequency clock, updating registers takes a while. Synchronization with the high frequency domain may be achieved with flags.

Prefix: COM

Board: Starter Kit EFM32-G8XX-STK Device: EFM32G890F128 (Gecko)

◆ LEUART0_IRQHandler()

void LEUART0_IRQHandler ( void  )

LEUART0 RX IRQ Handler.

RX Data Valid is handled as follows:
Copies one char from the RX-HW to the RX buffer.
'\r' and '\n' are considered as end of string character and COM_Received_Request() is called.
If the buffer is full, it is also handled as end of string.

TX Buffer Level is handled as follows:
If COM_TX_Busy_Flag is set
one char at a time is copied from the TX buffer to the TX-HW.
After sending the string COM_TX_Busy_Flag is cleared and the buffer level interrupt is disabled.
