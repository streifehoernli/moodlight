◆ PWR_conversion_shift

#define PWR_conversion_shift   12
Todo:
Define all the solution specific variables.

Avoid rounding issues in intermediate calculations by left-shifting input values first, then do the calculations and finally right-shifting the output values.
PWR_conversion_shift 12 is equivalent to fixed point calculation in Q19.12

Warning
: Avoid underflow and overflow of intermediate results!
Range of int32_t = -2'147'483'648 ... 2'147'483'647

◆ PWR_ACMP_IRQHandler()

void PWR_ACMP_IRQHandler ( void  )

ACMP interrupt handler.

Note
Be aware of the fact that ACMP0 and ACMP1 share the same interrupt service routine ACMP0_IRQHandler().
This implies that there is no ACMP1_IRQHandler().
ACMP1 is used for the touchslider and handled by ACMP0_IRQHandler() directly.
This PWR_ACMP_IRQHandler() handles the ACMP0 interrupt when it is called by ACMP0_IRQHandler() in touchslider.c
As part of an interrupt handler this is a time critical section.

TIMER0 interrupt handler.


Current measurements, control loops, adjusting PWM active times, etc. is done here for several solutions.

Note

CMSIS commands are used instead of EMLIB functions, because they run faster.

◆ PWR_conversion_input

const int32_t PWR_conversion_input[PWR_SOLUTION_COUNT] = { 0, 0, 0, 0, 0 }

Conversion factor from microcontroller-input user-interface-value. Use integer data types only and right-shifts instead of divisions for faster code execution (divisions are very time consuming).
PWR_value = (input_value * PWR_conversion_input) >> PWR_conversion_shift
